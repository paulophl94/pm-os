## How Pillars Are Used

- **Task alignment:** P1 tasks should tag a pillar: `- [ ] Task [Pillar: Name]`
- **Morning briefing:** Today's Three validated against pillars
- **Initiative tracking:** Each initiative in `pm-progress.json` should connect to a pillar
- **Weekly review:** Assess whether the week's work advanced pillar goals
- **Roadmap decisions:** Features that don't connect to a pillar need explicit justification

## Anti-Pillar Work

Work that doesn't connect to any pillar is not automatically wrong, but it should be questioned:
- **Operational necessity:** Server maintenance, bug fixes, compliance — tag as `[Operational]`
- **Ad-hoc requests:** Stakeholder asks that bypass the roadmap — tag as `[Ad-hoc]` and decide consciously
- **Exploration:** Early research into adjacent areas — tag as `[Exploration]` with time-box
