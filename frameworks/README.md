# Frameworks Reference

Strategic and product frameworks for use in analyses and documents.
These files are reference material — they are not frequently edited.

---

## Usage Map

| Framework | File | When to Use |
|-----------|------|-------------|
| **Rumelt Strategy Kernel** | [rumelt-strategy-kernel.md](rumelt-strategy-kernel.md) | Diagnose strategic problems, define guiding policy |
| **Gibson Biddle DHM** | [gibson-biddle-dhm.md](gibson-biddle-dhm.md) | Prioritize features by Delight, Hard-to-copy, Margin-enhancing |
| **Devil's Advocate** | [devils-advocate-strategy.md](devils-advocate-strategy.md) | Stress-test strategies and decisions, identify blind spots |
| **Impact Estimation** | [impact-estimation-framework.md](impact-estimation-framework.md) | Quantify expected feature impact, size opportunities |
| **SWOT Analysis** | [swot-analysis.md](swot-analysis.md) | Assess competitive position, map Strengths/Weaknesses/Opportunities/Threats |
| **Presentation Best Practices** | [presentation-best-practices.md](presentation-best-practices.md) | Prepare product presentations, stakeholder storytelling |

---

## By PM Context

| Situation | Recommended Framework |
|-----------|----------------------|
| Need to prioritize features | RICE (rule in `.cursor/rules/rice-analysis.mdc`) + Gibson Biddle DHM |
| Need to validate a strategy | Rumelt Strategy Kernel + Devil's Advocate |
| Need to estimate feature impact | Impact Estimation Framework |
| Need to map competitive position | SWOT Analysis |
| Need to present to leadership | Presentation Best Practices |
| Need to defend a decision | Devil's Advocate (to anticipate objections) |
